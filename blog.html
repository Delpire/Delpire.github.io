Blog #1 Introduction

In this blog I will be discussing the introduction to Gregory’s Game Engine Architecture which discusses some systems involved in a game engine. I will then discuss what some of these systems do, which I have have worked with, and which ones interest me the most.

The first chapter of Gregory’s Game Engine Architecture book goes through a variety of topics. He starts out by discussing his beginnings with the old Intellivision System and his purpose for writing the book, and then moves into a variety of topics: what is a game, what genres are out there, what platforms can be used, etc. Finally, he moves on to talk about various systems that are involved in or with a game engine such as the operating system, third-party SDKs, core systems, collision and physics, human interface devices, audio, multiplayer, networking, and more. None of these systems surprised me, and later I will discuss which I have had some experience with, and which I have interest in.

All of these systems are necessary when creating a game engine. The first is the operating system (OS). Most machines today have an OS installed on them, and any software written and run on this OS must follow some certain rules. For example, certain operating systems can only support certain languages. Also, the OS is a crucial part to communicating with the hardware, so this must be kept in mind when developing an engine. SDK’s are tools that already exist that can aid in creating a game engine. These can provide algorithms and data structures, graphics creation and rendering, physics, collision, animation, and much more. Human interface devices, are important systems for interacting with a game or game engine. This can be a mouse, keyboard, gamepad, or even something like the Oculus Rift, a headset for virtual reality. Audio is important as many games are improved with exciting music, sound, and cutscenes with voice over. Some games have multiplayer or online components, these systems are necessary for engines that wish to support these features.

In some way or another, I have touched all of these systems while developing software and many of them in the context of games. I took the game design class that was a prerequisite for this course, and we developed some basics in many of this systems to create simple 2D games. We touched on audio, collision and physics, taking input, and some other core systems. On other projects I have developed on the OS level and done some networking. In a competition at one of my internships, a friend and I developed a chat website making use of a third-party tool called socket.io that aided in web communication between clients. Socket.io could be used for a web based game engine as it is written in Javascript and intented for the web.

Networking is actually something I am incredibly interested in. I believe everything is moving towards the web platform as it is the most universal platform, existing on nearly all current devices (pcs, consoles, smartphones, tablets, etc.) When we develop our game engine, I would like to add some networking capabilities.

I am also interested in some of the core systems. I really enjoy writing algorithms, and I would like to learn about rendering graphics. I have loved math since highschool, so I think that many of the core systems will have some interesting challenges to tackle.

Overall, I am excited to begin working on implementing many of these game engine systems. I think that they will improve my skills as a programmer, as well as provide many interesting and fun problems to solve. In the end, I hope to get my hands on as many of these systems as possible.

Chris Delpire 2014
